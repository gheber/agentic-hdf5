# Paper Replication Test Suite

Test papers for the `hdf5-paper-replication` skill, ordered by difficulty.

## Test Papers

### 1. Easy: `easy_paper.pdf`
- **What**: Basic summary statistics (mean, std, min, max, threshold count) on a single HDF5 file
- **Data**: `sensor_readings_2024.h5` (local, generated by `generate_easy_data.py`)
- **Expected outcome**: Full replication, exact numeric match

### 2. Medium: `medium_paper.pdf`
- **What**: Linear regression of river discharge vs. catchment area across 12 stations
- **Data**: `river_discharge_2023.h5` (local, generated by `generate_medium_data.py`; paper claims Zenodo DOI which is fake)
- **Expected outcome**: Full replication. The agent should note the DOI is fake but find the local file.

### 3. Hard (synthetic): `hard_paper.pdf`
- **What**: Area-weighted global mean SST anomaly trend from NOAA ERSSTv5, 1980–2020
- **Data**: 492 NetCDF4 files from https://www.ncei.noaa.gov/pub/data/cmb/ersst/v5/netcdf/ (~1.5 MB each, ~740 MB total)
- **Expected outcome**: Agent should flag the 740 MB download size and confirm before proceeding. If approved, should replicate ~0.014°C/year trend. Minor numeric differences expected.

### 4. Real paper: `gw150914_tutorial.pdf`
- **What**: "The basic physics of the binary black hole merger GW150914" (LIGO/Virgo Collaborations, *Annalen der Physik* 2017). DOI: 10.1002/andp.201600209. Open access (CC-BY).
- **Data**: LIGO strain data in HDF5 from GWOSC (https://gwosc.org/events/GW150914/). Files are ~few MB each.
- **Analysis**: Newtonian approximation calculations on the strain time series: measure zero-crossing times to extract frequency evolution, read off peak gravitational-wave frequency (~150 Hz) and strain amplitude (~10⁻²¹), then use physics formulas to derive chirp mass (~37 M☉), luminosity distance (~400 Mpc), and radiated energy (~3 M☉c²).
- **Expected outcome**: Partial replication. The agent should be able to download the HDF5 data, apply a bandpass filter, measure zero-crossings, and compute the derived quantities. Results should be order-of-magnitude consistent with the paper. Exact match is not expected (Newtonian approximation, noise in data).
- **Why this is a good test**: The paper explicitly states its calculations are "straightforward enough that they can be readily verified with pencil and paper." It's a real published paper, with real public HDF5 data, doing real science.

## Generating Test Data

```bash
python generate_easy_data.py   # creates sensor_readings_2024.h5
python generate_medium_data.py  # creates river_discharge_2023.h5
```

Both scripts use fixed random seeds for reproducibility.
